# Vercel Deployment Guide for Tone Picker

## Prerequisites
1. GitHub repository with your code (follow github-setup.txt first)
2. Vercel account (free at https://vercel.com)
3. Your Mistral API key

## Deployment Steps

### Method 1: Vercel Dashboard (Recommended)
1. Go to https://vercel.com/dashboard
2. Click "New Project"
3. Import your GitHub repository (tone-picker)
4. Configure project settings:
   - Framework Preset: Other
   - Root Directory: ./
   - Build Command: npm run build
   - Output Directory: client/dist
   - Install Command: npm install

5. Add Environment Variables:
   - Click "Environment Variables"
   - Add: MISTRAL_API_KEY = your_api_key_here
   - Environment: Production, Preview, Development

6. Click "Deploy"

### Method 2: Vercel CLI
```bash
# Install Vercel CLI globally
npm install -g vercel

# Login to Vercel
vercel login

# Deploy from project root
vercel

# Follow the prompts:
# - Set up and deploy? Yes
# - Which scope? (your account)
# - Link to existing project? No
# - Project name: tone-picker
# - Directory: ./
# - Auto-detected settings? Yes

# Add environment variables
vercel env add MISTRAL_API_KEY

# Redeploy with environment variables
vercel --prod
```

## Configuration Files Added:
- vercel.json: Deployment configuration
- Package.json scripts should include build commands

## Build Script (add to package.json if missing):
```json
{
  "scripts": {
    "build": "tsc && vite build",
    "start": "node dist/server/index.js"
  }
}
```

## Environment Variables Required:
- MISTRAL_API_KEY: Your Mistral AI API key

## Post-Deployment:
1. Test all functionality on the live URL
2. Verify API endpoints work: https://your-app.vercel.app/api/health
3. Test tone adjustments with the live application
4. Update README.md with your live demo link

## Troubleshooting:
- Build fails: Check that all dependencies are in package.json
- API errors: Verify environment variables are set correctly
- 404 errors: Check vercel.json routing configuration
- CORS issues: Ensure your Express server handles CORS properly
